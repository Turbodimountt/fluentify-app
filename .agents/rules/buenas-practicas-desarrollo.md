Arquitectura y Clean Code (conventions.md)Propósito: Mantener el proyecto organizado y escalable según el stack definido.Backend (FastAPI): * Toda lógica de negocio debe residir en services/, no en los router/.Uso obligatorio de Pydantic v2 para validación de esquemas y SQLAlchemy 2.0 para la base de datos.Las respuestas de la IA deben implementarse mediante Streaming (EventSource/WebSockets) para cumplir con el RNF-01 (< 800ms).Frontend (Flutter):Uso estricto del patrón BLoC para la gestión de estados.Separación clara entre ui/ (widgets), domain/ (modelos) y data/ (repositorios/providers).Implementar diseño Responsive garantizando usabilidad desde 320px hasta Desktop.2. Seguridad y Privacidad (safety.md)Propósito: Cumplir con los requerimientos de seguridad y la normativa de protección de datos.Gestión de Secretos: Prohibido subir o escribir en código duro (hardcode) las API Keys de OpenAI, ElevenLabs o Google Cloud; usar exclusivamente variables de entorno.Privacidad de Voz: El audio capturado en el Modo Susurro debe procesarse en memoria y eliminarse inmediatamente tras la transcripción; nunca persistir archivos de audio.Acceso a Datos: Implementar RLS (Row Level Security) en PostgreSQL para asegurar que un usuario solo acceda a sus propios registros de conversación y progreso.3. IA y Prompt Engineering (ai_rules.md)Propósito: Asegurar que el Motor de IA se comporte según el modo seleccionado.Modo Susurro: Los prompts deben forzar un tono empático, cálido y limitar las correcciones a máximo 2 por turno para reducir la ansiedad.Modo Roleplay: La IA debe mantener el personaje del Contexto Profesional (ej. Medicina, Finanzas) y solo salir de él mediante el prefijo [Coach:] para dar feedback pedagógico.Feedback: Las correcciones deben seguir siempre la estructura: texto_original → correccion → explicación.4. Calidad y Pruebas (testing.md)Propósito: Garantizar la estabilidad del sistema antes del lanzamiento.Cobertura: El backend debe mantener una cobertura de tests unitarios y de integración ≥ 70%.Rendimiento: Validar que el análisis de trazos (escritura Kanji/Cirílico) responda en menos de 500ms para no romper la experiencia de usuario.Validación de Trazos: Utilizar MediaPipe para detectar los puntos de la mano antes de enviar el canvas al modelo de clasificación.
